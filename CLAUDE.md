# Jorge Sierra — Portfolio Website

## Project Goal
Build a premium single-page portfolio website for Jorge Sierra Guerra, Staff Growth PM and fintech product leader based in Guadalajara, Mexico. The site must feel like selected work — editorial, executive, credible — not a CV dump.

## Tech Stack
- HTML5 + Tailwind CSS (via CDN) + vanilla JavaScript
- Single `index.html` entry point for maximum portability
- Assets served from `./Assets/`
- Deployable on Vercel, Netlify, or GitHub Pages with zero config

## Design System
- **Palette**: Dark charcoal background (#0D0D0D or #111111) with warm off-white text (#F5F2EE), gold accent (#C9A84C), subtle slate for dividers (#2A2A2A)
- **Typography**: Inter (via Google Fonts) — weights 300, 400, 500, 700
- **Spacing**: Generous whitespace — breathe, don't crowd
- **Animations**: Subtle fade-in on scroll only. No flashy transitions.
- **Images**: Displayed as crisp screenshots in cards or lightbox modals
- **Feel**: Premium fintech/SaaS aesthetic — Bloomberg, Linear, Stripe docs influence

## Site Architecture

### Sections (in order)
1. **Hero / Cover** — name, tagline, one-liner positioning, CTA
2. **Executive Summary** — who he is, what he does, what makes him different
3. **How I Operate** — operating principles (3-4 pillars)
4. **Selected Work** — 8 project cases
5. **Credentials** — education, certs, mentoring
6. **Close + Contact**

## Asset Naming Convention
- `VC_*` → Vibe coding / built personally without engineering team
- `Consulting_*` → Advisory/consulting work (not direct employment)
- `[ProjectName]*` → Direct professional work

## Content Tone Rules
- English throughout (international audience)
- No CV speak: no "responsible for", "helped", "collaborated", "passionate about"
- Action-led, outcome-anchored
- Confidence without inflation
- Prudent language where outcomes are unclear ("drove", "shaped", "designed" vs "increased X by Y%")
- Use numbers only when confirmed in resume or brief
